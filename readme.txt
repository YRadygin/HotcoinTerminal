Идеи стратегий под депозит $1000
С малым счётом главные враги — комиссии и спред, поэтому стратегии с сотней сделок в день отпадают. Реалистичные варианты:

Grid-трейдинг (сетка) в боковике. Робот расставляет лимитные ордера на покупку/продажу с шагом, зарабатывая на колебаниях внутри диапазона. Идеально для текущего рынка, где BTC ходит $58–62k без направления. Скринер должен искать монеты в устойчивом боковике с хорошей волатильностью внутри диапазона.
Mean reversion (возврат к среднему). Покупка ликвидных монет на перепроданности (RSI < 30, отклонение от скользящей средней на 2+ стандартных отклонения) с целью на возврате. В условиях экстремального страха такие отскоки случаются часто, но нужен стоп-лосс — «перепроданное» может стать ещё перепроданнее.
Momentum-скринер по объёму. Автоматический поиск монет, где объём торгов вырос в 3–5 раз к среднему за 20 дней + пробой локального максимума. Пример из реальности: ALLO за день вырос с $0.24 до $0.36 при утроившемся объёме. Такие всплески ловятся именно скринером, глазами их не отследить. KuCoin
Событийная стратегия. Календарь разблокировок токенов (unlock — обычно давит цену вниз), листингов, крупных анонсов. Например, NEAR рос на ожиданиях крупного анонса 7 июля. Это полуавтомат: система собирает события, вы принимаете решение. KuCoin
DCA с фильтром. Скучно, но с $1000 разумно: регулярная докупка BTC/ETH только когда выполняются условия (страх ниже 20, цена ниже 200-дневной средней). Не даст быстрого плюса, но статистически сильна на дистанции.

Автоматический скринер — как это выглядит
Раз в N минут система тянет с API Hotcoin все тикеры → фильтрует мусор (объём меньше ~$1–5 млн/сутки, широкий спред — сразу вон, на них вы потеряете на проскальзывании) → считает индикаторы (RSI, ATR, отклонение объёма, положение в диапазоне) → присваивает каждой монете скоринг по каждой стратегии → топ кандидатов идёт в оповещение.
Роль Claude API
Правильное применение — не «предскажи цену» (это он не умеет, и никто не умеет), а:

анализ новостного фона по монете-кандидату: скринер нашёл всплеск объёма → Claude по свежим новостям объясняет, это листинг, партнёрство или памп-энд-дамп;
оценка тональности (sentiment) новостей и сводка «за/против» перед входом;
еженедельный отчёт по портфелю человеческим языком.

Оповещения
Самый практичный вариант на C# — Telegram-бот (библиотека Telegram.Bot): пуш прилетает мгновенно на телефон, туда же можно вывести кнопки «купить/пропустить», чтобы подтверждать сделки вручную. Это безопаснее полного автомата на старте. Альтернативы: e-mail, Windows-уведомления, если приложение десктопное.
Архитектура на C# (концептуально)
Worker Service (.NET 8) в фоне: модуль сбора данных с Hotcoin (REST + WebSocket для реального времени) → SQLite для истории свечей → библиотека индикаторов (есть готовая Skender.Stock.Indicators) → движок стратегий → модуль оповещений → опционально модуль исполнения ордеров. И обязательно: бэктестер на исторических данных и режим «бумажной торговли» (paper trading) минимум на месяц до реальных денег.
Риск-менеджмент, без которого всё остальное бессмысленно
На сделку — не более 2–5% депозита ($20–50), обязательный стоп-лосс, лимит дневных потерь (например, −5% депозита — робот выключается до завтра). С $1000 цель первых месяцев — не заработать, а не потерять, пока система обкатывается.
